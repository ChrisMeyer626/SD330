Christopher's Design Principles:

    Cognitive Design Principles

  • Ambition
  • Users should have a clear understanding of the system's objectives and how to achieve them.
  • Design the system to help users accomplish their tasks efficiently and effectively.
  • Allow users to feel in control and empowered when using the system.
  • Perception
  • Use contrasting colors and visual cues to draw attention to important elements making them stand out.
  • Use principles like proximity, similarity, closure, and continuity to organize information.
  • Consider the ease of pointing and selecting elements making it easier for users to interact with.
  • Memory
  • Minimize the need for users to remember specific details and provide cues/prompts to help.
  • Group related information or tasks into chunks or categories.
  • Use consistent and familiar terminology throughout the system.
  • Other
  • Use progress bars help users understand where they are within a multi-step process or task.
  • Implement a user-friendly onboarding process to introduce users to the system's features/functionalities.
  • Design the system to be aware of the user's context, such as location or time, and adapt accordingly.

    • Emotional/Accessibility/Error Principles

  • Provide immediate and clear feedback to user actions.
  • Use plain language and straightforward content that is easy to understand.
  • Provide informative and user-friendly error messages that help users.
  • Make it easy for users to recover from errors without losing their work.
  • Use sufficient color contrast to make content readable by people with visual impairments.

    • Key Principles

  • Performance
  • Reliability
  • Security
  • Simplicity
  • User-Centered

    Complexity Principles

  • Components in the system should be loosely coupled
  • Organize into distinct layers, each with a specific responsibility.
  • Implement thorough testing and automation throughout development.
  • Anticipate future scalability requirements and design with them in mind.
  • Hide complex implementation details behind simple, abstract interfaces.

    • Interface Principles

  • Create a clear and straightforward interface.
  • Maintain a consistent naming convention and design.
  • Think robust and handle unexpected or erroneous input gracefully
  • Make flexible enough to accommodate future changes and new features
  • Enable feedback and monitoring the use of the interface

    • Cultural/Gender Principles

  • Provide a culturally sensitive product.
  • Consider language preferences and communication styles that are inclusive.
  • Avoid using imagery, icons, or symbols that may be offensive or culturally inappropriate.
  • Ensure that your system is culturally accessible.
  • Identify and rectify potential biases in the system

    • Conceptional Principles

  • Define how data flows through the system and how different components interact with one another.
  • Design the system with ease of maintenance in mind.
  • Document the conceptual design, architecture, and interfaces to help understand structure and functionality.
  • Develop a system design complies with relevant laws and regulations.

    • Human Center Design Principles

  • Understand and empathize with the end-users by conducting research to find their needs.
  • Involve users in the design process through techniques like user interviews, surveys, and usability testing.
  • Create prototypes or mock-ups of the system to test and validate design ideas with users.
  • Develop user personas to represent different user groups.
  • Analyze the tasks and workflows that users need to accomplish with the system.
